My Adventure in Making a Blog

Date 12/28/2014 Author Kori  | Category : Computers

 

As I've told you, making a Blog from scratch is insanely difficult, and because of that I'm going to tell you how I did it.

  1. I Installed Ubuntu 14.04 Server Edition onto an old Dell Optiplex 760.
  2. Downloaded ssh and got it working on my server. To use it on my laptop I typed ssh (LAN IP address)
  3. I installed Virtualenv and created a virtual environment that was set to python 3.4
  4. While in my virtualenv I installed Django1.7.1
  5. To check if Django worked I installed w3m (a command line based browser) to see, which was good because at that point it didn't
  6. I set up Port Forwarding for my server which worked on Port 80
  7. Then I retried w3m and Django actually worked.
  8. Next I installed Nginx, but to do that I had to exit my virtualenv, it took me a while to figure out that the command to do that is deactivate.
  9. Back in my virtualenv, I installed Gunicorn .
  10. Got Sqlite3 setup with Django.
  11. Failed a lot trying to get Gunicorn and Nginx working, but eventually they did
  12. For the main blog work I mostly used a DjangoGirls Tutorial on how to make a blog.

Well those are all the steps I used to get my blog working, I tried more than that but those didn't really work. In the future I plan to do more Django tutorials so I can get a better understanding of how everything works and also add more features to this. I might even just throw this out and make an easier, more awesome website once I understand everything better. I did learn a lot from this experience though, I now have a better understanding of how files on Ubuntu get to the person who is viewing my website. I know Nginx is the way the internet connects to my files, it then deals with my static files, like CSS, while it passes the dynamic ones, html for example, to Gunicorn. While Django sets up the file structure and gives me a place to change the website before it deploys. Finally, I'm definitely more comfortable using command line because I had to make my whole website using nano(the command line file editor) and w3m and I had to make and delete directories, files, and navigate in and out of the file structure because it was really hard to make everything work together.
